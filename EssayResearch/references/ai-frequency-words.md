### 类别说明

| 类别 | 检测逻辑 | 典型问题 |
|------|---------|---------|
| 虚化修饰 | 副词"地"结尾的空泛修饰词，不提供量化信息 | "有效地提升了性能"——提升多少？基线是什么？ |

| 级别 | 触发条件 | 处理方式 |
|------|---------|---------|
| Info | 出现次数 ≤ 上限的 50% | 仅记录，不提示 |
| Warning | 出现次数 > 上限的 50% 且 ≤ 上限 | 提示作者关注，建议替换 |
| Error | 出现次数 > 上限 | 强制要求修改，标记为需修复项 |

## 二、句式多样性检测规则

### 2.1 段首结构重复检测

**规则**：连续 3 个段落的**首句**不得使用相同的句法结构。

**检测方法**：
1. 提取每个段落的首句。
2. 对首句进行浅层句法分析，归纳结构模板（如"主语+动词+宾语""介词短语+主谓""因为…所以…"）。
3. 若连续 3 个段落的首句结构模板相同，触发 Warning。

**示例**：
```
段落 A 首句："本研究提出了一种新的方法……"     → 结构：主语+动词+宾语
段落 B 首句："本文设计了一个新的框架……"       → 结构：主语+动词+宾语  ⚠️ 连续 2 次
段落 C 首句："本方案实现了一个新的系统……"     → 结构：主语+动词+宾语  ⚠️ 连续 3 次 → Error
```

**修复建议**：将段落 C 首句改写为不同的句法结构，例如：
- 条件句开头："在……条件下，本系统实现了……"
- 问句开头："如何在不损失精度的情况下……？本方案通过……"

### 2.2 段落长度标准差检测

**规则**：全文段落数 ≥5 时，段落长度（以字符计）的标准差应 > 30 字。

**检测方法**：
1. 统计每个段落的字符数（不含空白符）。
2. 计算标准差 σ。
3. 若 σ ≤ 30，触发 Warning，提示"段落长度过于均匀，可能缺乏详略节奏"。

**设计依据**：优秀的学术论文通常有长短交替的节奏——短段落用于强调或过渡，长段落用于详细论证。若所有段落长度趋同（如全部 150-170 字），说明行文机械，疑似模板化生成。

**示例**：
```
段落长度序列：[152, 155, 148, 150, 153, 149]
均值 = 151.2，标准差 ≈ 2.3  → 远低于 30  → Warning
```

**修复建议**：
- 将某个论证充分的段落扩展至 250+ 字。
- 将某个过渡性段落压缩至 50-80 字。
- 确保长短段交替出现。

## 三、清嗓子开头检测规则

### 3.1 规则定义

**清嗓子开头**（Throat-Clearing Opening）：指位于章节/段落开头、不含实质信息的冗余开场白。这类文字在口语或博客中尚可容忍，但在学术论文中属于噪声，应当删除。

**检测模式（正则匹配）**：

| 序号 | 模式 | 正则表达式（示例） | 说明 |
|------|------|-------------------|------|
| 1 | "为了更好地理解…" | `为了更好地理解[，,]` | 无信息量开场 |
| 2 | "在深入探讨…之前" | `在深入探讨.*之前[，,]` | 拖延正题 |
| 3 | "首先需要明确的是" | `首先需要明确的是[，,]` | 空洞引导句 |
| 4 | "让我们先来看" | `让我们先来看` | 口语化，不适合论文 |
| 5 | "值得一提的是" | `值得一提的是[，,]` | 无实际价值评价 |
| 6 | "在开始之前" | `在开始.*之前[，,]` | 元叙述，应删除 |
| 7 | "本章将…"（泛指） | `本章将.*介绍|本章将.*讨论` | 若后文已有小标题则冗余 |

### 3.2 处理逻辑

1. **检测**：扫描每个章节/段落的首句，匹配上述模式。
2. **标记**：匹配命中时，标记为 `[清嗓子开头-建议删除]`。
3. **建议**：提示作者直接删除该句，或将其中的有效信息融入后续正文。

### 3.3 示例

**修改前**：
> 为了更好地理解本系统的架构设计，在深入探讨具体实现之前，首先需要明确的是，本系统采用了模块化的设计理念。本系统的核心模块包括……

**修改后**：
> 本系统采用模块化设计理念，核心模块包括……

## 四、标点模式控制

### 4.1 破折号控制

**规则**：全文破折号（——）出现次数 ≤ 5 次。

**设计依据**：破折号在学术论文中用于补充说明或转折，但过度使用会打断阅读节奏，显得随意。5 次是单篇论文的合理上限。

**检测方法**：统计全文中文破折号"——"的出现次数。超过 5 次触发 Warning，建议将部分破折号替换为逗号或括号。

### 4.2 感叹号正文禁用

**规则**：正文（不含代码注释、图表标注）中**禁止使用感叹号（！）**。

**设计依据**：学术论文应保持客观、克制的语调。感叹号传递的情感色彩与学术写作规范冲突。正文中的感叹号一律视为格式错误。

**检测方法**：扫描正文段落中的中文感叹号"！"和英文感叹号"!"。命中即标记为 Error，要求删除或替换为句号。

**例外区域**（不检测）：
- 代码块内的注释
- 引用原文中的感叹号（引号内）
- 图表标题/脚注中的强调标注

### 4.3 其他标点规范

| 标点 | 规范 | 说明 |
|------|------|------|
| 省略号 | 使用"……"而非"..." | 中文省略号为六点 |
| 引号 | 中文用""'' | 不使用""'' |
| 顿号 | 用"、"分隔并列词语 | 不用逗号代替顿号 |
| 冒号 | "："后接分述内容 | 避免冒号后直接句号 |
| 分号 | 长句并列分句间使用 | 分号后首字母不大写（中文无大小写问题） |

## 五、检测流程集成

本文件定义的所有检测规则在 EssayResearch 写作后处理阶段按以下流程执行：

```
正文文本
  │
  ├─ 1. AI 高频词扫描（§一）
  │     └─ 输出：告警列表（词 / 出现次数 / 级别）
  │
  ├─ 2. 句式多样性检测（§二）
  │     ├─ 2.1 段首结构重复检测
  │     └─ 2.2 段落长度标准差检测
  │
  ├─ 3. 清嗓子开头检测（§三）
  │     └─ 输出：建议删除句列表
  │
  ├─ 4. 标点模式控制（§四）
  │     ├─ 4.1 破折号计数
  │     ├─ 4.2 感叹号禁用
  │     └─ 4.3 其他标点规范
  │
  └─ 汇总报告 → 写入 quality-report.json
        └─ 包含：Error 数、Warning 数、逐项明细
```
